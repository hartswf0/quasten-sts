Bogost Ian Persuasive Games 2007

1. Problem Statement

  1. Bogost, Ian - Persuasive Games (2007)

Focused Problem Statement: Analyzing serious games like a climate change simulator (L1 Machine Layer) only through their narrative or visual elements obscures their primary persuasive mode, identified by Bogost (L3 Framing) as "procedural rhetoric." The game's rule-based simulation of climate dynamics (L0 Reality) implicitly argues for specific viewpoints (L4 Embedded Ideology) through the consequences experienced by the player (L2 Human Use), a rhetorical function masked if only surface features are considered.

2. Core Model

9. Bogost, Ian - Persuasive Games (2007)

  • MODEL: Argues that video games function as persuasive rhetoric not just through narrative or visuals, but primarily through their underlying rules and systems ("procedural rhetoric"), which model real-world processes and implicitly argue for particular perspectives or ideologies.
  • Index:
    • ◻ Video Games → ◻ Rules/Systems (Procedurality)
    • ◻ Procedurality → ◻ Modeling Real-World Processes
    • ◻ Modeling Real-World Processes → ◻ Procedural Rhetoric
    • ◻ Procedural Rhetoric → ◻ Persuasion/Argument (for Ideologies/Perspectives)

4. Spatiotemporal Framework

9. Bogost, Ian - Persuasive Games (2007)

  • ◻ SPATIOTEMPORAL UNIVERSE: Contemporary culture where video games are a significant medium.
    • → [Contains] → ◻ FIELD: Video games as rhetorical systems.
      • → [Has] → ◻ DOMAIN: Game rules, mechanics, simulation models, player interaction patterns × Cultural context of play.
      • → [Projects] → ◻ SIGNAL: Player actions, system responses, game outcomes, emergent behaviors, interface elements. (ℝᵏ representing game state variables, player choices, success/failure metrics).
      • → [Admits] → ◻ PERTURBATION: Player strategies, game modifications (mods), different hardware platforms, critical analysis of games.
    • → [Observed Through] → ◻ WINDOW: Rhetorical theory, procedurality studies, software studies, game studies.
      • → [Defines] → ◻ OBSERVATION SCALE: Focus on the underlying rules and processes (procedures) of games as the primary site of meaning and argument.
      • → [Delimits] → ◻ CONTEXT BOUNDARY: Privileges systemic expression over narrative or visual rhetoric, analyzes "serious games" and mainstream games alike.
      • → [Generates] → ◻ SAMPLING LATTICE: Analysis of specific game mechanics, simulation algorithms, rule sets, examples of political/advertising/simulation games.
    • → [Manifests As] → ◻ PHENOMENON: Procedural Rhetoric – The practice of using computational processes and rule-based systems (like those in video games) to make arguments, persuade players, and model ideologies about how the world works (or should work).
  • ◻ ENGINE: Procedural Rhetorical Analysis.
    • → [Applies] → ◻ TRANSFORM: Interpreting game rules and mechanics as implicit arguments or models of reality.
      • → [Operates On] → ◻ PATCH: Specific game rules, player interactions with systems, simulation outcomes.
      • → [Produces] → ◻ INVARIANT: The underlying persuasive argument embedded in the game's procedural logic; the ideology represented by the simulation.
      • → [Requires] → ◻ CONTRACTION: Focusing on the systemic level of rules and processes, potentially bracketing player interpretation or narrative overlay.
    • → [Constructs] → ◻ REPRESENTATION: Analyses demonstrating how specific games make procedural arguments (e.g., about economics, politics, social systems).
    • → [Optimizes] → ◻ RECURSION: Applying the concept of procedural rhetoric to a wide variety of games and computational systems.
      • → [Embeds] → ◻ MODEL: Theories of rhetoric, computation, simulation, systems thinking.
      • → [Projects] → ◻ METRIC: Clarity/effectiveness of the procedural argument, complexity of the simulation model, ideological bias (qualitative/structural).
      • → [Seeks] → ◻ FIXED POINT: A clear articulation of the specific argument being made by a game's procedures.
  • ◻ METAMODEL: Theory of Procedural Rhetoric. Governs the analysis of games as persuasive systems. Binds Computational Processes/Rules DOMAIN ↔ Rhetorical Arguments/Ideological Models CODMAIN.
  • Distinctions & Crossings:
    • ◻ MODEL (The game's rule system/simulation) | ◻ REALITY (The real-world system being modeled/argued about) → ◻ MEASUREMENT ARTIFACT (The game's necessary simplifications and biases are artifacts of its modeling process).
    • ◻ TIME-LIKE (Player progression through the game, system dynamics unfolding) | ◻ SPACE-LIKE (The rule-space, the set of possible game states). The CAUSAL CONE represents how player actions within the rules lead to consequences.
    • ◻ CONTINUOUS (Real-world processes) | ◻ DISCRETE (Game rules, state changes, player inputs) → ◻ SAMPLING THRESHOLD (The game discretizes reality into rules and quantifiable variables).
  • Operations: Observe gameplay and rules. Encode mechanics as rhetorical statements. Convolve player actions (PATCH) with game rules (KERNEL) to produce outcomes (FEATURE). Entangle game systems with real-world ideologies. Might Forget non-procedural aspects (e.g., aesthetics, narrative) if focus is too narrow.
